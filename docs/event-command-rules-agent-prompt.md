# 事件指令规则子代理任务契约

本契约用于把事件指令参数里的玩家可见文本整理成导入规则。执行者只按本文件、CLI 输出、工作区文件和任务说明工作，不读取项目源码、数据库或程序内部对象。

## 输入

- 只读文件：`<工作区>/event-commands.json`。
- 任务参数：`<项目目录>`、`<工作区>`、`<游戏标题>`。
- 可参考命令结果：主代理提供的 `export-event-commands-json --json` 输出摘要。

## 逻辑

- 按事件指令编码分组检查 `parameters` 中的字符串叶子、字符串数组和嵌套对象。
- 只选择玩家会在对话、菜单、选择、提示、任务、状态、说明或插件命令显示内容中看到的自然语言文本。
- 排除资源路径、文件名、脚本、公式、数字、颜色、坐标、布尔值、枚举值、开关名、内部标识、调试字段和纯配置键。
- 需要把同一编码下不同形态区分开时，用 `match` 精确限定参数索引和值；`match` 的键必须是参数索引字符串。
- 没有过滤条件时，`match` 写 `{}`。
- 插件命令的常见结构可以参考 `parameters` 数组位置判断，但必须以当前导出的 `event-commands.json` 为准；不要把某个编码或插件名当成所有游戏通用答案。
- 字符串像 JSON 时，可以解析后继续判断里面的字符串叶子；最终 JSONPath 仍必须指向 `event-commands.json` 中真实存在的字符串叶子。
- 不确定字段先看同一指令内相邻参数、对象键名和值的语义；仍不确定时排除，并在完成报告写明风险。

## 输出

- 唯一可写文件：`<工作区>/event-command-rules.json`。
- 顶层必须是对象，格式为 `{指令编码字符串: [{match, paths}]}`。
- `match` 必须是对象，键为参数索引字符串，值为对应参数的精确字符串值。
- `paths` 必须是 JSONPath 字符串数组，路径从 `$['parameters']` 开始，并命中字符串叶子。
- JSONPath 必须使用括号路径语法；禁止使用 `$.xxx` 点号路径。
- 合法空结果是 `{}`，表示已确认没有可导入事件指令参数规则。

中性格式示例：

```json
{
  "<指令编码>": [
    {
      "match": {
        "0": "<用于区分形态的参数值>"
      },
      "paths": [
        "$['parameters'][3]['<玩家可见文本字段>']"
      ]
    }
  ],
  "<另一个指令编码>": [
    {
      "match": {},
      "paths": [
        "$['parameters'][<参数索引>]"
      ]
    }
  ]
}
```

## 校验

写完后建议主代理运行：

```powershell
<att-mz> validate-event-command-rules --game <游戏标题> --input <工作区>/event-command-rules.json --json
```

如果校验失败，只修 `<工作区>/event-command-rules.json` 后重跑同一校验命令；不要绕过 CLI 导入。

## 完成报告

完成时报告：

- 改动文件。
- 命中的指令编码、规则数量和路径数量。
- 是否为空结果；为空时说明检查范围和空结果理由。
- 主要排除类别。
- 未解决风险。
- 建议主代理运行的校验命令。
