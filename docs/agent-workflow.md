# Agent 翻译流程概览

本文档给用户快速理解流程。Agent 的完整执行规程在 `skills/rpg-maker-translation/SKILL.md`。

## 总流程

```text
准备输入
→ Agent 读取 SKILL.md
→ 检查项目和游戏
→ 扫描自定义控制符
→ 准备并导入术语表、插件规则、事件指令规则
→ 执行翻译
→ 生成质量报告
→ 质量通过后写回
→ 用户实机抽查
```

## 用户负责什么

- 提供项目目录。
- 提供游戏根目录。
- 提供外部临时目录。
- 在终端环境变量中配置模型服务地址和 API Key。
- 决定是否允许最终写回。
- 在 Agent 遇到阻断问题时做选择。
- 写回后进行实机抽查。

## Agent 负责什么

- 按 `skills/rpg-maker-translation/SKILL.md` 执行。
- 检查项目环境和游戏结构。
- 判断自定义控制符规则是否完整。
- 产出并导入术语表、插件规则和事件指令规则。
- 执行翻译和质量报告。
- 遇到阻断问题时暂停并反馈。
- 在质量报告通过且用户允许后写回。

## CLI 负责什么

- 读取标准 RPG Maker MZ 数据。
- 管理每个游戏的数据库。
- 提取正文、插件文本和事件指令文本。
- 调用模型并缓存译文。
- 校验占位符、日文残留和长文本宽度。
- 写回游戏文件并建立留档。

## 阻断时应该发生什么

如果项目目录不存在、游戏数据不可解析、模型不可用、自定义控制符未覆盖、外部规则导入失败、翻译多轮失败、质量报告有阻断错误或用户未授权写回，Agent 应停止当前阶段并向用户报告。

报告应包含：当前阶段、已执行内容、阻断原因、关键证据、建议下一步和需要用户确认的信息。
