# 把本项目交给外部 Agent

本文档给用户使用。Agent 的完整执行规程在 `skills/att-mz/SKILL.md`，用户只需要准备输入、启动 Agent，并要求 Agent 按 Skill 执行。

## 1. 用户需要准备什么

- 项目目录：`<项目目录>`
- 游戏根目录：`<游戏根目录>`
- 外部临时目录：`<外部临时目录>`
- 模型服务地址环境变量：`RPG_MAKER_TOOLS_LLM_BASE_URL`
- 模型 API Key 环境变量：`RPG_MAKER_TOOLS_LLM_API_KEY`
- 是否允许最终写回游戏文件

模型地址和 API Key 不要写进 CLI 参数。推荐在启动 Agent 的终端里设置环境变量。

```powershell
$env:RPG_MAKER_TOOLS_LLM_BASE_URL = "<模型服务地址>"
$env:RPG_MAKER_TOOLS_LLM_API_KEY = "<API_KEY>"
```

## 2. 给 Agent 的启动提示词

把下面这段交给外部 Agent：

````text
请使用 <项目目录> 中的 RPG Maker MZ 翻译工具处理 <游戏根目录>。

输入：
- 项目目录：<项目目录>
- 游戏根目录：<游戏根目录>
- 外部临时目录：<外部临时目录>
- 模型环境变量已在当前终端配置：是/否
- 是否允许最终 write-back：是/否
- 任务类型：首次翻译/二次翻译/只做诊断

执行要求：
- 先读取并严格遵守 skills/att-mz/SKILL.md。
- SKILL.md 是唯一执行手册；docs/ 里的用户文档只作为说明，不作为执行规程。
- 不要把模型地址和 API Key 写进 CLI 参数、文档、日志说明或临时文件。
- 如果项目、游戏、模型、占位符规则、外部规则、质量报告或写回权限存在阻断问题，必须暂停并向我报告。
- 不要在自定义控制符规则未确认覆盖前启动 translate。
- 如果数据库中的 name-context、plugin rules 或 event-command rules 为空，必须由你自己导出和分析；游戏中实际存在对应内容时生成导入文件、导入数据库并验收通过；确认游戏本身没有对应内容时记录结论并继续。
- 导出结果为空、插件没有玩家可见文本、事件指令参数没有玩家可见文本时，不要编造规则；记录确认结论即可。
- 如果你不确定结构含义，必须暂停向我反馈，不要把不确定当作无内容。
- 如果你支持子代理，可以并行处理 name-context、plugins 和 event-commands；但你必须等待全部子代理完成，并负责最终复核、导入数据库、验收和清理临时文件。
- 导出和导入必须使用本项目 CLI；中间分析可以使用你自己的搜索、编辑、脚本或子代理能力。
- 三类规则数据导入并验收通过，或确认游戏本身没有对应内容后，删除本轮导出和分析产生的临时文件。
- 不要在 quality-report 存在阻断错误时执行 write-back。
- 所有临时导出和 Agent 产物写入 <外部临时目录>。
- 主流程需要的数据必须通过 import-* 命令写入数据库。
- 完成后报告已执行命令、质量报告结论、是否写回，以及建议我实机抽查的重点。
````

## 3. 用户如何判断 Agent 做得对不对

Agent 应该先检查项目和游戏，再扫描自定义控制符。只要数据库中的术语表、插件规则或事件指令规则为空，Agent 必须自己导出对应文件并分析内容。游戏中实际存在对应内容时，生成导入 JSON、导入数据库并验收；确认游戏本身没有对应内容时，记录确认结论并继续。导出结果为空、插件没有玩家可见文本、事件指令参数没有玩家可见文本，都是可接受结论。翻译后必须生成质量报告。只有质量报告无阻断问题，并且用户允许写回时，才可以写回游戏文件。

如果 Agent 在缺模型、找不到项目、游戏数据不可解析、强制规则数据未检查、翻译连续失败或质量报告有阻断问题时继续硬跑，说明它没有按 Skill 执行。

二次翻译时，Agent 应先检查已有数据库、规则、缓存、运行状态和质量报告，不应把任务当成全新游戏无脑重跑。已有译文缓存会复用；只有规则缺失、规则过期、文件结构变化或质量报告提示风险时，才重新导出和分析对应数据。

## 4. 用户什么时候需要介入

这些情况需要用户确认或补充信息：

- Agent 找不到项目目录。
- Agent 找不到游戏根目录。
- 游戏目录不是 RPG Maker MZ 标准结构。
- `data/*.json` 缺失、加密、乱码或无法解析。
- 模型环境变量未配置。
- 模型连接失败、认证失败或被内容审查拒绝。
- 自定义控制符是否需要保护无法判断。
- 翻译连续多轮失败且错误数量没有明显下降。
- 是否允许最终写回游戏文件。

## 5. 推荐使用方式

先在游戏副本上运行，不要直接在唯一原始目录上首跑。写回后进行实机抽查，重点检查开场剧情、菜单界面、插件界面、地图名、名字框、长文本和含控制符文本。
