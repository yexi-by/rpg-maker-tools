# RPG Maker MV/MZ 游戏翻译常识参考

本文件是 A.T.T MZ Skill 的按需参考资料。只有在判断 RPG Maker MV/MZ 引擎协议、控制符、事件指令、数据库字段、插件参数或 Note 标签时才读取；不要把全文塞进模型提示词、子代理任务单或面向用户的交付报告。

## 使用边界

- 本文件提供的是 RPG Maker MV/MZ 常识和翻译判断原则，不替代 CLI 校验结果。
- 当前游戏的真实规则仍以 `prepare-agent-workspace` 导出的工作区文件、CLI `--json` 输出、已导入规则和游戏目录内容为准。
- 如果本文件与当前游戏插件文档或工作区候选冲突，先按当前游戏证据判断，再把不确定点报告给用户。
- 面向模型的正文翻译 prompt 只提供当前文本、输出协议、质量要求和必须原样保留的控制符，不解释本文件里的内部处理流程。

## 标准控制符常识

MV/MZ 的标准消息控制符必须按完整格式判断。只看到反斜杠加字母前缀时，不等于已经看到完整合法控制符。

| 控制符 | 标准含义 | 翻译处理 |
| --- | --- | --- |
| `\V[n]` | 显示第 `n` 号变量值 | 原样保留完整片段 |
| `\N[n]` | 显示第 `n` 号角色名 | 原样保留完整片段 |
| `\P[n]` | 显示队伍第 `n` 位成员名 | 原样保留完整片段 |
| `\G` | 显示货币单位 | 原样保留完整片段 |
| `\C[n]` | 切换文字颜色 | 原样保留完整片段 |
| `\I[n]` | 显示第 `n` 号图标 | 原样保留完整片段 |
| `\{` | 放大文字 | 原样保留完整片段 |
| `\}` | 缩小文字 | 原样保留完整片段 |
| `\\` | 显示反斜杠字符 | 原样保留完整片段 |
| `\$` | 打开金钱窗口 | 原样保留完整片段 |
| `\.` | 短暂停顿 | 原样保留完整片段 |
| `\|` | 较长停顿 | 原样保留完整片段 |
| `\!` | 等待玩家输入 | 原样保留完整片段 |
| `\>` | 后续文字立即显示 | 原样保留完整片段 |
| `\<` | 取消立即显示 | 原样保留完整片段 |
| `\^` | 显示后不等待 | 原样保留完整片段 |
| `\PX[n]` | 设置窗口内文字 X 坐标 | 原样保留完整片段 |
| `\PY[n]` | 设置窗口内文字 Y 坐标 | 原样保留完整片段 |
| `\FS[n]` | 设置字号 | 原样保留完整片段 |

判断要点：

- `\N[1]` 是完整标准控制符；裸 `\N` 不是标准角色名控制符。
- `\V`、`\P`、`\C`、`\I`、`\PX`、`\PY`、`\FS` 也需要对应参数，裸前缀通常不能当完整标准控制符。
- 裸 `\G` 是标准控制符，因为货币单位控制符本来不带参数。
- 小写 `\n` 常见于 JSON 字符串、插件参数或转义文本里的字面量换行，不能被 `(?i)\\N\d*` 这类宽规则误匹配。

## 事件指令常识

MV/MZ 地图和公共事件常见存放在 `data/*.json`。翻译判断时只需要理解可见文本来源，不需要记住项目数据库结构。

| 指令编码 | 常见用途 | 翻译关注点 |
| --- | --- | --- |
| `101` | 显示文本窗口设置 | MZ 通常可在 `parameters[4]` 存名字框；MV 标准结构通常没有第 5 个参数 |
| `401` | 显示文本正文行 | 正文翻译主来源 |
| `102` | 显示选项 | 选项文字是玩家可见文本 |
| `405` | 滚动文本正文行 | 正文翻译来源 |
| `356` | MV 插件命令 | 常见为单个插件命令字符串，必须按导出样本判断 |
| `357` | MZ 插件命令 | 参数结构由插件决定，必须按工作区导出判断 |

判断要点：

- 事件指令编码是引擎结构常量，可以作为规则分析依据。
- MV 默认分析 `356`，MZ 默认分析 `357`；CLI 显式 `--code` 会覆盖默认值。
- MZ `357` 的参数里可能混有插件名、命令名、显示名、参数对象、资源名、脚本和玩家可见文本，不能整块都翻译。
- MV `356` 常见为单个字符串，可能同时包含命令名、参数和值；只能按当前 `event-commands.json` 判断哪些字符串叶子是玩家可见文本。

## 数据库字段常识

数据库里的角色名、技能名、物品名、装备名、敌人名、状态名、系统术语、地图显示名等可能在 UI、战斗、菜单和剧情里复用。

处理原则：

- 字段译名表负责精确写回数据库名称、系统词、地图显示名，以及 MZ 标准名字框等字段。
- MV 说话人通常来自插件、文本控制符、事件指令字符串或自定义协议，不能默认补写 `101.parameters[4]`。
- 正文术语表只负责给正文翻译提示统一译名，不负责写回字段。
- 字段值如果包含包装符号、名牌触发标记或插件协议，字段译名表要保留或重组这些可运行片段，不能指望正文术语表自动修复。

## 插件与 Note 标签常识

MV/MZ 游戏常通过插件扩展文本来源。`plugins.js`、插件命令、Note 标签都可能包含玩家可见文本，也可能只是机器协议。

判断玩家可见文本的常见信号：

- 自然语言说明、菜单项、按钮名、任务提示、帮助文本、状态说明、物品详细说明。
- 地图上方显示的名牌、气泡文字、提示语、任务目标。
- 插件 UI 中会展示给玩家看的窗口标题、列表项、确认/取消文案。

必须排除的常见机器协议：

- 文件路径、图片名、音频名、动画名、图标索引、颜色、坐标、数字、布尔值、枚举值。
- JavaScript 脚本、公式、条件表达式、开关名、变量名、内部 key、资源 ID。
- 只用于插件判定或绑定的标签值，哪怕里面含有日文字符，也不能直接当玩家文本翻译。

## 翻译语境常识

MV/MZ 汉化不是逐行替换日文，还要保留游戏运行协议和 UI 可读性。

- 对话要优先自然中文语气，同时保留控制符、等待、颜色、图标和变量显示。
- UI 文本要短、清楚、可扫读，避免过长导致窗口放不下。
- 技能、物品、状态和系统词要稳定统一，避免同一概念在菜单和剧情中出现多个译名。
- 角色名、称号、地名和组织名要先看上下文，避免机械音译。
- 如果原文是作品名、Staff 名、品牌名或玩家需要识别的专有名词，可以保留日文，但必须通过日文保留例外规则限制到具体文本。

## 工具异常和游戏异常的区分

- 完整标准控制符被报告为未覆盖：优先怀疑工具扫描或内置规则异常，停止并报告工具问题。
- 裸 `\N`、`\V` 等缺参数片段被报告为未覆盖：优先怀疑源文本不完整、插件自定义协议或扫描提示需要人工判断，不要直接定性为工具异常。
- 自定义规则通过覆盖扫描但校验拒绝：通常是规则太宽、模板不合法或误匹配常见转义，先修规则。
- 当前游戏插件文档明确说明某个裸片段是合法插件协议：按插件协议写精确自定义规则，并在最终报告说明这是当前游戏插件规则，不是 RPG Maker 标准控制符。
